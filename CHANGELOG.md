# 更新日志

本文档用于记录项目的重要变更。

## 0.62a - 2026-07-22

> `0.62a` 为流程表单与人工审批工作流版本；npm 与 Cargo 包版本仍标识为 `0.2.0-alpha.0`。

### 新增

- 新增流程表单类型。创建流程表单时会自动生成与该表单一一对应的审批流程，并在应用导航、表单列表和应用设置中以独立图标及类型标识展示。
- 新增流程工作流运行时：支持提交已保存记录、发起流程实例、生成待办任务、同意、拒绝和已审批记录反审；实例、任务和操作历史均持久化，并保留所用流程版本。
- 流程编辑器新增审批、执行、抄送和结束节点；审批/执行节点可配置处理人及会签方式，流程可复用数据操作和 HTTP 请求节点。
- 流程表单记录新增审批状态、实例状态、当前审批节点和提交人等可配置表格字段；记录页提供提交与反审操作。
- 新增通用工作流画布、节点注册表、属性抽屉和图校验基础组件，覆盖起始节点、断链、循环、孤立节点及节点配置校验。
- 新增应用导航项排序接口，以及表单创建类型、应用所有者头像、记录创建者头像和所属组织等接口字段。

### 调整

- 自动化定义区分事件触发流与流程流：事件触发执行不会误触发流程流，流程流固定绑定其流程表单及 `form_submit` 触发事件，并限制可用节点类型。
- 应用创建者改为当前登录用户，应用列表与卡片可展示所有者头像；表单记录列表补充创建者身份和组织信息。
- 表单运行时拆分关联表单、成员选择和子表单表格组件，关联记录与成员目录改为缓存加载；子表单支持按配置分页显示。
- 重构表单视图管理、应用资源加载与自动化编辑器的共享逻辑，收敛重复请求和状态处理。
- OpenAPI 契约、前端 SDK 及类型定义同步更新，以覆盖流程表单、导航排序和补充的资源元数据。

### 修复

- 修复流程任务仅按显示名称校验处理人的问题，现优先按用户 ID 校验，并为待办查询建立对应索引。
- 修复流程定义编辑可能修改绑定表单或触发事件的问题，并阻止普通自动化创建流程提交触发器。

## 0.56a - 2026-07-21

> `0.56a` 为 Agent 可控执行、资源运行时与应用工作区权限细化版本；npm 与 Cargo 包版本仍标识为 `0.2.0-alpha.0`。

### 新增

- Agent 配置档案新增创建表单草稿与保存 Schema 草稿的受控能力；Agent 可在获授权的当前应用范围内创建表单，但仍不能发布、删除或修改既有资源。
- Agent 配置档案新增创建应用、创建表单、创建自动化 3 个能力开关；当前运行时已实际接通创建表单与保存 Schema 草稿，创建应用和自动化开关暂未绑定对应工具。
- Agent 运行时接入已绑定的 Skill 指令、知识库内容检索和插件工具清单，并记录解析后的资源审计信息。
- Skill 支持以独立 `.yaya-skills/<package>/SKILL.md` 文件包读写；知识库支持维护正文内容并按用户问题检索相关片段。
- 默认 Agent 配置自动补齐表单设计、表单草稿填写、自动化审查 3 个内置 Skill，以及表单组件与自动化审查 2 个知识库；其中表单设计顾问预授权应用列表、表单列表、表单 Schema、创建表单草稿、保存 Schema 草稿 5 个平台工具；同时提供默认禁用的 HTTP JSON 插件模板。
- 插件支持 JSON Manifest 声明工具，并可在无人工确认要求时受控调用 HTTP 工具端点；新增 HTTPS、私有地址、重定向和响应大小等安全限制。
- 新增平台工具目录、Skill 文件读写、授权集合及应用默认导航入口接口。
- 新增关联表单字段：设计器可选择关联范围、显示字段、筛选条件与回填映射；运行时可读取关联表单记录，选中记录后自动回填目标字段。
- 新增成员选择字段的设计与运行时配置，支持按本地、钉钉、企业微信或飞书来源，以及全部成员、指定角色、指定成员范围进行单选或多选。
- Agent 可在未提交的表单运行时页面协助填写字段值；该模式仅回写浏览器草稿，明确禁止代替用户提交数据。
- 新增表单设计器对关联表单配置的发布前校验，以及关联表单字段的列宽调整能力。
- 自动化字段映射新增可搜索字段和函数的公式编辑器，支持插入常用数学、逻辑与文本函数。
- 应用设置支持选择默认导航入口；新建表单会在尚无默认入口时自动成为默认入口。

### 调整

- RBAC 扩展为平台菜单分组授权及应用内细粒度权限；应用开发明确拆分为编辑应用信息、创建分组、集成自动化、应用设置、应用发布 5 项，并补充表单开发和视图开发入口；前端导航会按授权结果隐藏不可访问项。
- 权限检查从设置页延伸到应用顶部导航、应用侧栏、表单设计器、表单操作和 API 代理；无表单开发权限时，设计器提供明确的受限状态。
- Agent 助手、资源配置档案与平台设置界面补充能力开关、资源内容、插件 Manifest 和平台工具展示，并优化会话与页面上下文传递。
- Agent 助手由居中弹窗调整为右侧抽屉，增加工具调用进行中/完成状态、按最新会话排序、智能自动滚动和回到最新消息操作，并按 Agent 配置权限控制配置入口。
- 自动化编辑器改用抽屉式属性面板，新增 Schema 查看器并收紧映射控件布局；运行时公式与表单设计器的编辑反馈同步优化。
- 组织架构接口和页面改为返回、展示、筛选部门直属成员；角色页和权限页增加身份来源筛选与更清晰的授权状态展示。
- 设计器完善关联表单、成员、子表单和容器的属性配置与布局约束，允许子表单在分组容器中按容器宽度布局，并保持子表内部字段的布局限制。
- 表单运行时补充关联记录选择、成员候选加载、子表行关联字段回填等交互；应用设置、表单工作区、首页、侧栏和设置中心同步调整布局与状态反馈。
- OpenAPI 文档、前端生成 SDK、类型定义和后端路由同步扩展，供前端代理调用新增的授权、平台工具、Skill 文件及默认入口接口。

### 修复

- 修复本地凭据实体、迁移、OpenAPI 契约和生成客户端在新增配置字段后的同步问题。
- 修复应用导航默认入口切换、表单运行时异步加载、钉钉组织同步与多个 API 请求的授权和错误处理边界。
- 修复组件布局计算对未知字段类型的处理，并避免关联表单未完成显示配置时发布无效 Schema。

## 0.5a - 2026-07-20

> `0.5a` 为身份认证、RBAC 和表单数据工作区迭代版本；npm 与 Cargo 包版本仍标识为 `0.2.0-alpha.0`。

### 新增

- 新增本地账号密码登录、JWT 会话与钉钉 OAuth 扫码登录；钉钉账号可按配置在首次登录时自动创建平台用户。
- 新增本地用户与本地角色的创建、编辑、删除和角色分配能力，支持维护用户资料、多个邮箱地址与账号状态。
- 新增 RBAC 权限中心，可为角色配置平台设置、应用和表单的显示、编辑、创建、删除等授权；系统管理员保留全量权限。
- 新增认证与授权中间件，覆盖应用、表单、自动化和平台设置接口，并按授权结果过滤应用导航。
- 新增表单自定义视图的服务端持久化及 CRUD，表单记录接口支持分页查询。
- 新增字段大纲页面与接口，可按应用浏览表单、字段、Schema 版本及动态物理表信息。
- 新增后端 `/openapi.json` 契约导出与前后端联合启动脚本，支持在生成客户端前合并 OpenAPI 基线契约。

### 调整

- 身份源设置精简为钉钉配置，补充清理同步数据、AccessToken 管理和 OAuth 回调链路。
- 登录页支持记住密码、自动登录和钉钉授权跳转；侧边栏改为账户菜单，并重构首页、应用中心和设计器入口布局。
- 首页改为连接真实应用数据的工作台，展示最近创建、最近访问、待处理入口及应用统计；最近访问记录保存在浏览器本地。
- 应用中心统一使用页面头部与卡片布局，应用内侧栏、主题切换菜单和账户菜单同步收紧尺寸与交互层级。
- 设置端重构用户、角色、组织架构、身份源与权限页面，补齐本地用户资料、邮箱、角色分配和授权配置的管理交互。
- 表单运行时的视图配置从浏览器本地存储迁移至后端，记录表格补充分页加载、视图切换、列宽计算和请求失败状态处理。
- 设计器画布替换通用字段预览为轻量化预览，优化布局、字段描述、组件箱和工作台面板的显示与编辑反馈。
- 前端 API 代理统一转发浏览器会话令牌，并逐步切换到基于生成客户端的后端 SDK 代理，减少各路由重复的转发逻辑。
- 表单删除流程改为事务化清理关联记录、视图、导航、自动化定义、版本与运行日志。

### 性能优化

- 表单记录查询支持数据库级分页与总数统计，避免数据量增大时一次性加载全部记录。
- 设计器画布与工作台侧栏引入 `memo` 和数据缓存：预计算字段占用单元格，减少拖拽、缩放和属性编辑时的重复扫描与无关面板重渲染。
- 设计器直接操作期间跳过布局动画，并临时关闭画布与记录详情背景的毛玻璃滤镜，降低频繁重绘与合成开销。
- 表单运行时预先构建分组子字段索引，并通过稳定回调和上下文引用降低字段输入时的组件更新范围。
- 公式执行缓存编译结果、字段依赖关系和循环引用分析，避免每次值变更重复解析同一公式。

### 修复

- 修复部分表单与应用侧边栏请求失败时的状态处理，授权不足时不再保留不可访问的导航项。
- 修复表单删除后遗留自动化关联数据的问题。
- 修复运行时公式字段在依赖更新后的计算与错误反馈链路，并改进表格文本宽度估算以减少列宽跳动。
- 修复多处异步请求缺少异常处理、受控状态同步和 React Effect 触发的更新警告。
- 将 OpenAPI 生成文件加入忽略规则，避免自动生成产物进入版本变更记录。

## 0.3a - 2026-07-15

> `0.3a` 为 Alpha 阶段的存储模型与工作区体验迭代版本；npm 与 Cargo 包版本仍标识为 `0.2.0-alpha.0`。

### 新增

- 新增按表单编译的动态混合存储：已发布表单拥有独立物理数据表，标量字段映射为带类型的数据库列，复杂字段保留在扩展 JSONB 中。
- 新增子表单子表与同步机制，支持子表单行数据独立落库，并在表单 Schema 更新时回填对应数据。
- 新增 `form_storage_definitions` 存储定义表，保存表单物理表、字段映射和已编译 Schema 版本。
- 表单设计器新增字段索引配置面板，支持为可映射的标量字段配置 B-tree 索引。
- Agent 注册配置补充插件、Skill、知识库定义 CRUD，并支持在 Agent 配置档案中绑定这些资源。

### 调整

- 表单记录仓储改为存储无关接口，表单运行时和自动化节点均通过动态表读写记录。
- 表单创建、发布和 Schema 更新流程会同步存储计划；表单与应用删除时会清理对应的动态表和存储定义。
- 表单详情工作区重新组织工具栏：筛选、显示列、排序靠近表单编辑；记录操作使用带提示的图标按钮；搜索改用 HeroUI `SearchField`。
- 更新应用侧栏、内容区和表单导航布局，并新增卡片毛玻璃语义色；亮暗主题背景渐变与卡片表面调整为更易区分的层次。

### 修复

- 移除运行时共享 `form_records` 表及其实体，避免所有表单记录继续堆积在共享 JSONB 结构中。
- 存储计划编译时拒绝重复字段 ID，避免字段列映射冲突。
- 修复表单 Schema 更新后子表单投影未回填的问题。
- 表格区域改为按需显示纵向滚动条，移除右侧始终预留滚动条造成的背景缝隙；操作列会占用剩余可用宽度。

### 当前限制

- 动态表建表、索引和子表单触发器尚缺少真实 PostgreSQL 的端到端集成测试。
- JSONB 仍有意用于复杂字段扩展数据、子表单行数据和存储计划元数据，不再作为共享表单记录的唯一持久化模型。
- 插件、Skill 和知识库当前完成配置管理与绑定，运行时插件加载、Skill 执行和向量检索仍待实现。

## 0.28a - 2026-08-04

> `0.28a` 为中期迭代版本；npm 与 Cargo 包版本仍标识为 `0.2.0-alpha.0`。

### 新增

- 新增 Agent 多级管理体系：
  - 模型提供商（Provider）配置与 CRUD 页面 `/settings/model-providers`
  - Agent 配置档案（Config Profile）与 CRUD 页面 `/settings/agent-profiles`

  - Agent 人格（Persona）预设（默认人格、业务分析师、低代码实施顾问）
  - 插件（Plugin）、技能（Skill）、知识库（Knowledge Base）定义与管理 API
- 新增 Agent 注册表（Agent Registry）后端持久化，支持按平台/应用/业务范围解析 Agent 配置，兼容旧版 Agent 设置自动迁移。
- 新增身份源管理：
  - 钉钉集成：AccessToken 获取与刷新、部门同步、用户同步的后端接口

- 新增 Agent Markdown 渲染组件（`react-markdown` + `remark-gfm`）。
- 新增 `xlsx` 依赖用于 Excel 批量导入导出。
- 新增表单 GET API 端点（`GET /api/forms/:form_uuid`）。
- 新增设计器校验模块（`designer-validation.ts`）。
- 新增设置页面子路由：模型供应商、Agent 管理、配置档案、身份源、组织架构、插件、Skills、用户、角色。

- 后端设置模块扩展：新增身份源配置读写、Agent 注册表读写、Agent 配置按范围解析。
