# 更新日志

本文档用于记录项目的重要变更。

## 0.28a - 2026-08-04

> `0.28a` 为中期迭代版本；npm 与 Cargo 包版本仍标识为 `0.2.0-alpha.0`。

### 新增

- 新增 Agent 多级管理体系：
  - 模型提供商（Provider）配置与 CRUD 页面 `/settings/model-providers`
  - Agent 配置档案（Config Profile）与 CRUD 页面 `/settings/agent-profiles`
  - Agent 定义（Agent Definition）管理页面 `/settings/agents`
  - Agent 人格（Persona）预设（默认人格、业务分析师、低代码实施顾问）
  - 插件（Plugin）、技能（Skill）、知识库（Knowledge Base）定义与管理 API
- 新增 Agent 注册表（Agent Registry）后端持久化，支持按平台/应用/业务范围解析 Agent 配置，兼容旧版 Agent 设置自动迁移。
- 新增身份源管理：
  - 钉钉集成：AccessToken 获取与刷新、部门同步、用户同步的后端接口
  - 平台用户体系，支持本地和钉钉双身份来源
  - 身份源设置页面 `/settings/identity-source`，配置保存后立即生效
- 新增身份与权限数据模型与数据库迁移：
  - `organization_units` 组织单位表
  - `iam_users` / `iam_external_identities` 用户与外部身份表
  - `iam_roles` / `iam_user_roles` 角色与用户角色关联表
  - `iam_organization_memberships` 用户组织成员关系表
  - Agent session 绑定 agent_id 迁移
- 新增子表单（Subform）组件运行时渲染：
  - 表格视图，支持行新增、删除、复制、上移/下移排序
  - 斑马纹、分割线、边框三种主题
  - 序号列、操作列显示控制，自定义列宽与冻结
  - 批量导入、导出 Excel、批量删除、子表单标题过滤
  - 布局模式（auto/fixed）、每页行数、最大行数限制
- 新增表单运行时 `onValuesChange` 回调和 `valuePatch` 外部值同步能力。
- 新增 Agent Markdown 渲染组件（`react-markdown` + `remark-gfm`）。
- 新增 `xlsx` 依赖用于 Excel 批量导入导出。
- 新增表单 GET API 端点（`GET /api/forms/:form_uuid`）。
- 新增设计器校验模块（`designer-validation.ts`）。
- 新增设置页面子路由：模型供应商、Agent 管理、配置档案、身份源、组织架构、插件、Skills、用户、角色。

### 调整

- Agent 设置页面 `/settings/agent` 改为自动重定向到 `/settings/agents`。
- 设置导航从 4 项扩展为 13 项，支持多级分组与独立路由。
- 设计器工作台、画布、侧边栏、字段属性面板、页面属性面板、布局系统大幅重构，增强交互体验和视觉一致性。
- 设计器新增调试事件通道、页面级 Agent 上下文配置等能力。
- 全局 CSS 主题变量、玻璃拟态样式、布局高度和滚动容器全面优化。
- 运行时表单分组容器边距缩减（`p-4` → `p-1`），Radio/Checkbox 选项文本字号调整为 12px。
- Agent 助手面板重构，支持模型切换、上下文统计和更丰富的会话交互。
- 后端设置模块扩展：新增身份源配置读写、Agent 注册表读写、Agent 配置按范围解析。

### 修复

- 修复表单预览和运行时下拉单选/下拉复选组件（SelectPreview / MultiSelectPreview）从非受控改为受控模式，解决选择其他选项后显示不更新的问题。
- 修复后端 Agent 流式响应中 `assistant_message` 插入数据库的 `Result` 未处理问题，添加 `error!` 日志和 `message.persist_failed` SSE 事件通知。
- 移除运行时表单页面对本地 `testSchema.json` 的硬编码依赖，改为完全从 API 动态加载 Schema，并添加 schema 为 null 时的加载状态保护。
- 新增 `.yaya-agent-registry.json` 和 `.yaya-identity-settings.json` 到 `.gitignore`，防止敏感配置被误提交。

### 当前限制

- 知识库与 Skills 后端定义已完成，前端页面仍为功能占位，尚未接入向量检索和 Skill 加载器。
- 插件机制目前为管理端的定义配置，尚未实现运行时插件的加载与执行。
- 子表单当前仅支持运行时渲染，设计器端拖拽和属性配置尚未完成。
- 身份源的用户/角色列表当前为只读查询，尚未接入权限校验中间件。
- 项目仍处于 Alpha 阶段，接口、数据结构与交互可能继续调整。

## 0.2a - 2026-07-13

> `0.2a` 为早期预览版本；npm 与 Cargo 包版本使用语义化标识 `0.2.0-alpha.0`。

### 新增

- 新增平台级 Agent MVP：
  - 支持 OpenAI Compatible 模型供应商、API Base URL、对话模型、Embedding 模型、Temperature 与最大执行步骤配置。
  - 新增全局 Agent 助手入口、会话列表、历史消息与 SSE 流式输出。
  - 新增只读工具，可读取应用表单列表、表单草稿 Schema、自动化列表与自动化流程图。
  - 新增 Agent 会话、消息、运行与步骤审计表：`agent_sessions`、`agent_messages`、`agent_runs`、`agent_run_steps`。
- 新增平台设置子路由与左侧导航：
  - `/settings/database`
  - `/settings/agent`
  - `/settings/knowledge`
  - `/settings/skills`
- 新增应用设置页面，提供基础设置、表单设置、应用管理员、权限和数据工厂等配置入口；当前除应用名称加载外，其余配置为前端占位。
- 新增系统、浅色、深色三种主题模式与主题切换菜单。
- 新增表单默认值公式能力、公式函数定义与运行时公式求值支持。
- 新增应用详情代理接口，支持应用设置页读取当前应用名称和基础信息。
- 新增基于 `@dnd-kit/core` 的导航与设计器拖拽交互支持。

### 调整

- 重构全局页面外壳、侧边栏、首页、应用列表与应用内导航，统一玻璃拟态主题变量和响应式布局。
- 重构表单设计器工作台、组件箱、画布、预览、字段属性与页面属性面板，增强历史版本读取和表单名称编辑体验。
- 优化自动化列表与编辑器布局，增强节点配置、条件分支、字段映射、表达式引用和运行日志展示。
- 设置页面改为固定占用主区域剩余高度；左侧导航和右侧设置面板分别独立滚动。
- 数据库和 Agent 配置改为后端本地持久化，敏感字段不回显到浏览器。
- 后端增加 Agent 数据库迁移、实体、路由、模型执行器与只读工具模块。

### 修复

- 修复设置内容较多时面板超出屏幕、内容较少时未撑满剩余高度的问题。
- 修复 Agent 启动器 Hook 依赖与本地消息 ID 生成导致的 ESLint 错误。
- 移除 README 和后端默认连接字符串中的明文数据库密码。
- 将数据库与 Agent 本地设置文件加入 Git 忽略规则，避免 API Key 或数据库凭据被误提交。
- 修复多处布局高度、滚动容器、主题颜色、表单预览和自动化编辑交互问题。

### 当前限制

- Agent 当前仅允许调用只读工具，不会创建、修改、发布或删除业务数据。
- 知识库与 Skills 页面当前为功能占位，尚未接入向量检索和 Skill 加载器。
- 项目仍处于 Alpha 阶段，接口、数据结构与交互可能继续调整。

## 2026-06-29

### 新增

- 新增集成自动化模块，支持应用级自动化列表、自动化编辑页与工作流保存。
- 新增自动化触发器节点、查询节点、新增数据节点、更新数据节点、删除数据节点、HTTP 请求节点、条件分支节点。
- 新增自动化版本快照能力，支持版本列表查询与历史版本恢复。
- 新增自动化结构化持久化表：
  - `automation_flows`
  - `automation_flow_versions`
  - `automation_nodes`
  - `automation_edges`
- 新增自动化运行日志能力，支持查看流程运行记录、节点执行记录、输入输出载荷、耗时与重试来源。
- 新增失败处理能力：
  - 整个流程重头触发
  - 错误节点断点重试
- 新增表单记录运行时编辑、删除接口，并接入自动化触发链路。
- 新增前端 Next.js 代理接口，用于转发自动化详情、版本、运行日志与重试相关请求。

### 调整

- 将自动化编辑器重构为更适合复杂节点配置的暗色工作区布局，属性面板仅在选中节点时显示，最大宽度限制为 `500px`。
- 将工作流名称、说明、启停状态与版本管理上移到编辑器左上角头部区域，移除触发器节点中的工作流级配置项。
- 调整自动化画布交互，点击空白区域时隐藏属性配置，减少右侧配置区常驻造成的干扰。
- 调整条件分支节点的数据源策略，改为可使用全部上游节点产生的数据，而不是仅使用直接上级节点。
- 调整条件分支配置模型，支持：
  - 所有数据均可通过
  - 按条件规则进入
  - 按公式定义进入
- 调整触发器节点配置逻辑：
  - 初始节点名称、说明不可编辑
  - 变更字段仅在编辑前/编辑后事件下展示
- 调整应用内导航，在集成自动化页面隐藏不必要的表单导航干扰项。
- 更新 OpenAPI 描述与前端生成客户端，补齐本轮自动化相关接口的类型定义。

### 修复

- 修复非触发器节点错误显示工作流配置的问题。
- 修复节点属性抽屉在选择部分配置值后意外关闭的问题。
- 修复拖拽连线到空白画布后未弹出新增节点对话框的问题。
- 修复自动化编辑器布局撑高页面、导致浏览器出现多余滚动条的问题。
- 修复运行日志接口在前后端版本不一致时返回空响应，前端表现为 `503` 的问题；现在会返回明确的重启后端提示。
- 修复自动化运行日志查看路径 `404` 排查链路，确认问题来源为旧版后端进程未重启。
- 修复新增字段配置中公式输入框、多条数据来源节点映射等错误交互。
- 修复触发器节点以外的节点仍可编辑不该暴露字段的问题。

### 说明

- 当前自动化运行已支持触发事件：
  - `before_create`
  - `after_create`
  - `before_update`
  - `after_update`
  - `before_delete`
  - `after_delete`
- 当前自动化执行已支持节点类型：
  - 获取单条数据
  - 获取多条数据
  - 新增数据
  - 更新数据
  - 删除数据
  - HTTP 请求
- 当前错误节点重试已经从“整条流程重新执行”调整为“基于失败节点输入快照的断点恢复执行”。
- 若本地仍访问不到运行日志接口，需要确保后端已在本轮代码更新后重新启动。

### 后续优化

- 为自动化运行日志增加后端分页、筛选、统计汇总与日志保留策略。
- 为 HTTP 请求节点补充更完整的响应状态、响应头与错误上下文展示。
- 统一自动化接口到前端生成 API Client，减少运行日志相关页面对手写 `fetch` 的依赖。
- 为自动化执行链路增加事件来源标记、递归深度限制与循环触发防护，避免未来出现自动化自触发套娃风险。
- 补齐更完整的条件表达式编辑器、字段映射编辑器与运行前校验能力。

### 文档

- 删除重复职责的 `README.log`，统一由 `README.md` 承载项目说明，由 `CHANGELOG.md` 承载版本更新记录。
- 更新 `README.md`，补充集成自动化、运行日志、自动化存储结构、本地运行说明与下一步规划。
